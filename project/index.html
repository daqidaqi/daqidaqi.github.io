项目信息（易卷 - Esurvey）

项目介绍

一款发布，收集用户信息的问卷调查系统

体验地址

  1. 移动端网页：http://app.cymgc.top
  2. 后台管理：http://admin.cymgc.top

项目背景

  1. 易班平台没有可以发布，收集信息的问卷平台，接入第三方问卷服务费用较高
  2. 学校的部门招聘采用纸质填写信息，可以使用这个问卷系统收集面试者的信息
  3. 也可以通过这个平台发送面试情况信息通知，避免了短信通知被拦截导致没有收到的情况（拓展）

项目意义

  1. 方便用户在易班平台填写问卷
  2. 方便对信息进行分析和可视化
  3. 方便收集用户信息（如面试者信息，各种活动报名信息）
  4. 可以作为反馈平台收集意见（拓展）
  5. 方便报名后查看活动（面试）进度（拓展）

项目亮点

  1. 免费平台
  2. 网页平台，可以接入易班平台，方便找到和使用
  3. 获取易班用户信息，学校学生信息，定制化需求设计（比如有辅导员列表可选，选择填写范围）
  4. 定制化设置不同用户填写规则（可以根据不同的场景设计规则，如报名时每人只能报名一次，每个人只能隔两周报一次或者每个年级不同的人报名的次数和时间不同等等）

技术选型

移动端网页前端

技术及版本 作用 原因
Uniapp 移动端网页开发框架 跨端，后期想转微信小程序方便
Uview 2.x 适配 uniapp 组件库 原本想用 uview-plus（支持 vue3），不过文档（实例是Vue 2的示例）和生态没有uview2.x 的好，再加上我对 Vue3 也不太熟悉，之前也用过 uview 写过小程序，另一方面uview的功能组件属性比 uni-ui 丰富，综上考虑选择 uview2.x

后台管理前端

技术及版本 作用 原因
React 17.0.0 作为前端开发框架 组件库比较丰富
Umi 3.5.0 前端开发扩展封装框架 丰富的插件使用
Ant design pro v5 作为开发组件库使用 开发脚手架很方便
Pro component 1.1.15 作为组件库拓展 页面组件库，更方便，使用 React 开发主要原因

后端

技术及版本 作用 原因
Spring Boot 2.7.0 主要开发框架 现在主流的开发框架，比较熟悉，项目本身体量不大，不需要使用微服务
Lombok 注解式代码生成 节省开发时间

部署

技术及版本 作用 原因
宝塔面板 管理服务器 可视化操作服务器
Nginx 部署前端，反向代理 比较熟悉的网关，性能高

其他

技术及版本 作用 原因
form-generator 表单生成器 支持 React，文档详细
