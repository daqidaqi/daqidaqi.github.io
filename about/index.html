凉茶小栈

自我介绍

小伙伴们，大家好，我是凉茶，加入星球希望可以和大家一起做项目，一起进步。

高中

高中时期，在高中的时候我就对编程感兴趣，虽然没有明确的计划，但是我知道我想学计算机。那时候也查了一些资料，基本都是在推荐 python ,于是我就去自学 python。我不记得我在哪里看的课程了，好像是在微信里面刷到的一套课，做一个小东西，当时跟着那门课学习，学习了 python 的基础语法，也不算入门，当时那个构造方法就给我搞蒙了，还有 this 也看不懂，不过也算是给我打开编程的大门。

高考那时候考得比较差，考到一所二本学校，刚进入大学的时候，会觉得失落之类，感觉我不应该在这个平台，不过现在我就释然了，无所谓，在哪都能学编程。我的专业也与计算机有关，课程中有学计组，计网，数据结构这些，也算半个科班吧。

大一

大学期间，没有上大学的时候，我对大学是充满幻想的，课程少，轻松，还有各种各样的社团，各种各样的实验室，各种各样有趣的活动和比赛。现在看来，我觉得最好的资源还应该是图书馆，我也经常去借书来看，不过一般都是先看微信阅读，要付费了就去借图书馆的书。以前我都是很功利的看书，不管是高中读的书还是大学读的书，我都没读进去，尽管我能背诵，能考试，但就是不能理解，在以考试为目标的前提下， 读书就变成了一件很痛苦的事情，就像最近看的《如何阅读一本书》，里面说的方法很多都是语文课上的内容，什么略读，粗读，现在重新学习这些知识，我才真正理解和使用起来。诶，扯远了，我继续介绍。

大学刚开始，我就想着加一个实验室或者部门，最好是学技术的，我也不考虑当班干，一切以做自己喜欢做的为主。之后我加了一个部门，部门里面学的是 java，因此我也将经历转到了 java ，python 的话到现在还处于高中阶段，不过后面看到 python 的代码，也能理解一些，核心的不变，只是语法不同，api 不同。部门面试是学 html，css，然后笔试，在跟着 pink 老师学习后，我也能够写一些简单的静态页面了，真的那时候成就感满满。在部门中，部长们给我们提供了一些学习资料，还有解答平时的一些疑问，我意识到还是得自学，一开始我以为部门可以给我培训之类，不够也不可能做到，b站一些视频都是几十个小时，怎么能给你培训，我也意识到大学好像并不是我想象的那样，能在大学里面学到的知识是有限的。之后，我也把大部分经验放在b站自学上。大学，给我的感受是，一个资源平台，给你充分时间学习的地方，大学课程不深但挺广，这应该是为了让学生接触到更多的东西，找到自己喜欢的，然后自学，深入学习（仅我对学校的理解）。

大一上，我开始学习 javaSE，那段时间真的很劝退，一是不知道学了这些东西要干嘛，二是学了也记不住焦虑，有时候我也在想我真的喜欢写代码吗，不过最后还是学完了，对于是不是真的感兴趣，我觉得如果一件事情你想去做，做了，又颓废了，但最后还是继续去做，那就可能真的是你感兴趣的，也能够一直做下去。有段时间我去学尤克里里，做了，颓废了，然后没有然后哈哈。又跑了，我们继续看经历。

大一寒假，学完 javaSE，我继续学习 javaWeb，maven，学习 Servlet，JDBC,这时候我也能写一些网页，然后交互了，不过我没有停留太久，在写了一些 demo 之后就继续学习后面框架了。在学放寒假的时候，我基本学完了 Servlet，也做了一个前后端交互的一个特别简单的系统，能登录注册，crud。寒假，我继续学习 SpringBoot，Mybatis。

在大一下，我学完 SpringBoot ，并做了一个项目，后台管理系统，没有权限管理，用的是 vue + springboot，不过功能不全，也只能算是 demo 吧。我学习了 linux，学习部署，我也成功把网站部署到服务器上，用的是 java -jar，学习到这里，我发现学习比以前快了，也较轻松了，没有向入门阶段那样“痛苦”（相对而言，也不算特别轻松）。

大一暑假，我看到了学长的学校查课表，查成绩的小程序，我也心血来潮，想搞小程序，做一个资源共享小程序，蓝图还有添加学校地图，导航，积分系统，签到打卡….，我学习 Oss 对象存储，学习 uniapp 写小程序，后台用的是ruoyi，本来也想直接写的，不过有一种无从下手的感觉。这也是我现在想加星球的原因，我也想在星球学习一下项目，做一个框架，然后学习更多的项目，抽离里面的业务代码，封装得模块，加工一下集成到框架里面。小程序上线了，后台管理系统也部署了，学会了买服务器，域名，备案，ssl证书，用 docker 部署（期间操作不当导致服务器被拿去挖矿，只能重装系统），不过现在没有维护了，我觉得需要一次大重构，ruoyi我用的不明白，里面很多是我用不到的，反而增加成本。

大二

大二上，我继续学习 springcloud，nacos，gateway… 不过也只是学完知识，没有写项目。我感觉现在好像其实有点乱，学太快了，巩固太少了，基本都是 demo，没有一个完整的项目。我的学习基本都是看黑马学的，里面的项目我也有做，不过做到一半就停滞了，bug好多，版本冲突好难处理，跟着视频敲，他运行成功，我已运行，bug满天飞，我满天找。不过，想想也正常，视频不过用来引流的，能把之前的项目放出来给我吗免费观看就很良心了，毕竟免费，还要什么自行车。加入星球，我主要是想学一下项目，同时找一些志同道合的小伙伴。

大二寒假，也就是现在，这个寒假因为疫情比较长，可以学更多的内容，不过我的显示屏在学校，笔记本有点小，比较难受。快哭了，图书馆很少去自习，都是去借书，也是因为没有显示屏，写代码看视频都很难受，切来切去，有点闪眼，不知道有没有小伙伴知道有什么好的办法解决，你们也是这样吗。

目前情况，有计组，计网的基础，没有做过完整的项目，我打算跟着鱼皮哥做，至少bug会少点。同时 javaSE 的基础也不是很好，比如多线程，和反射，io我都忘的差不多，主要是平时用的不多。我学习的话都是先快速的过一遍，我个人觉得编程不是看一遍就会了，就行了，肯定要二次学习，深入学习，很多东西我都看完视频，也只是浅浅了解。目前，我在学习算法和做项目，不过安排的不是很合理，两个同时进行，消耗精力有点多，我是看视频学的数据结构和算法，没有刷过 leetcode，主要是确实难，耗时，又想学算法，又想做项目，目前在做用户中心，我是不是应该先专注于一个，小伙伴有什么建议吗？

好了，希望接下来我能和大家一起进步，同时，也祝大家新年快乐！耶
